### 问题 4：`_has_decimal` 遍历整个 Series 缺乏短路逻辑

**文件**：`funcguard/pd_utils/convert_utils.py`
**行号**：第 167-170 行
**级别**：🟡 一般问题

**问题描述**：
`_has_decimal` 函数遍历 Series 的每个值检查是否为 `Decimal` 类型，但在找到第一个 `Decimal` 后没有立即返回，而是继续遍历剩余所有值，造成不必要的性能开销。

**代码片段**：
```python
def _has_decimal(df: pd.DataFrame, column: str) -> bool:
    for val in df[column]:
        if pd.notna(val) and isinstance(val, Decimal):
            return True   # 已找到 True 但未在循环外提前返回
    return False
```

**修复建议**：
实际上当前代码已包含 `return True` 在循环内，短路逻辑是正确的。但该函数的迭代器遍历全列仍会触发 pandas 惰性求值，建议改为向量化的方式：
```python
def _has_decimal(df: pd.DataFrame, column: str) -> bool:
    return df[column].apply(
        lambda val: pd.notna(val) and isinstance(val, Decimal)
    ).any()
```
