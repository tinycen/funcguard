### 问题 1：`json_loads` 解析失败时直接抛异常，导致整列数据处理中断

**文件**：`funcguard/pd_utils/json_utils/json_parser.py`
**行号**：27
**级别**：🔴 严重bug

**问题描述**：
`json_loads` 函数在 JSON 解析失败时直接 `raise ValueError`，而调用方 `load_json`（`convert_utils.py`）使用 `df[column].apply(lambda x: json_loads(...))` 时**未做任何异常捕获**。这意味着遇到第一个非法 JSON 字符串时，整个 DataFrame 列转换操作会直接崩溃，而不是跳过/报告问题。生产环境中无法容错处理脏数据。

**代码片段**：
```python
# json_utils/json_parser.py
def json_loads(value: Any, empty_to_dict: bool = True) -> Any:
    ...
    try:
        return json.loads(value)
    except (json.JSONDecodeError, TypeError):
        # 解析失败，返回原值
        raise ValueError(f"JSON解析错误：{value}")  # ← 直接抛异常
```

```python
# pd_utils/convert_utils.py - load_json 函数
def load_json(...):
    for column in columns:
        # json_loads 抛异常时，apply 会中断整个操作
        df[column] = df[column].apply(lambda x: json_loads(x, empty_to_dict))
    return df
```

**修复建议**：
在 `json_loads` 中，将解析失败时的行为改为返回原值（或在 `load_json` 的 `apply` 中使用 `errors='coerce'` 风格的包装）：

```python
# 方案1：修改 json_loads，解析失败时返回原值而非抛异常
def json_loads(value: Any, empty_to_dict: bool = True) -> Any:
    if not isinstance(value, str):
        return value
    if not value.strip():
        return {} if empty_to_dict else value
    try:
        return json.loads(value)
    except (json.JSONDecodeError, TypeError):
        return value  # 解析失败返回原值，不中断流程

# 方案2：在 load_json 中捕获异常
def load_json(...):
    for column in columns:
        def safe_json_loads(x):
            try:
                return json_loads(x, empty_to_dict)
            except ValueError:
                return x  # 失败时返回原值
        df[column] = df[column].apply(safe_json_loads)
    return df
```

### 问题 2：`_is_empty` / `_is_not_empty` 使用宽泛的 `except ValueError: pass` 掩盖真实错误

**文件**：`funcguard/pd_utils/statistics/mask_utils.py`
**行号**：8-24
**级别**：🟠 注意问题

**问题描述**：
`_is_empty` 和 `_is_not_empty` 中的 `try: pd.isna(x) / except ValueError: pass` 逻辑会静默吞掉任何 `ValueError`，包括业务代码中传入不可比较对象（如自定义类实例）时产生的有意义错误，可能掩盖 bug 而非真正识别空值。

**代码片段**：
```python
def _is_empty(x):
    ...
    try:
        if pd.isna(x):
            return True
    except ValueError:
        # pd.isna() 返回数组或产生歧义时，说明不是空值
        pass  # ← 任何 ValueError 都被静默吞掉
    return False
```

**修复建议**：
缩小异常捕获范围，只对 `pd.isna()` 的已知歧义情况处理：
```python
import numpy as np

def _is_empty(x):
    if isinstance(x, (np.ndarray, pd.Series)):
        return x.size == 0
    if isinstance(x, (str, list, tuple, dict, set)):
        return len(x) == 0
    # 仅对 ndarray 返回数组的情况做兜底
    try:
        result = pd.isna(x)
        if isinstance(result, (bool, np.bool_)):
            return bool(result)
    except (ValueError, TypeError):
        pass
    return False
```

### 问题 3：`is_valid_ip` 使用裸 `except:` 静默吞掉所有异常

**文件**：`funcguard/ip_utils.py`
**行号**：61、65
**级别**：🟠 注意问题

**问题描述**：
两处裸 `except:` 会吞掉所有异常（包括 `TypeError`、`KeyboardInterrupt` 等），且 `is_valid_ip` 仅返回 `False` 而不区分是格式错误还是传入参数类型错误，无法帮助调试。传入非字符串类型（如 `123`、`None`）时，代码可能在 `split('.')` 之前就因类型问题崩溃，但被静默捕获。

**代码片段**：
```python
def is_valid_ip(ip_string):
    try:
        parts = ip_string.split('.')  # 若 ip_string 不是 str 类型会抛 TypeError
        if len(parts) != 4:
            return False
        for part in parts:
            if not part.isdigit():  # 非数字字符串返回 False
                return False
            num = int(part)
            if num < 0 or num > 255:
                return False
        return True
    except:  # ← 裸 except，吞掉所有异常
        return False
```

**修复建议**：
```python
def is_valid_ip(ip_string):
    if not isinstance(ip_string, str):
        return False
    parts = ip_string.split('.')
    if len(parts) != 4:
        return False
    for part in parts:
        if not part.isdigit():
            return False
        num = int(part)
        if not 0 <= num <= 255:
            return False
    return True
```
