**文件**：`funcguard/printer.py`

**级别**：🔴 严重bug

**问题描述**：

**代码片段**：
```python
def print_progress(idx: int, total: int, message: str = "") -> None:
    # ...
    percent = int(idx / total * 100)  # total=0 时崩溃
    bar = '█' * (percent // 2) + '-' * (50 - percent // 2)
```

**修复建议**：
在计算前增加守卫条件：
```python

    if total <= 0:
        print(f"\r进度: |{'-' * 50}| 0% (0/0) {message}", end='', flush=True)
        return
    percent = int(idx / total * 100)
    bar = '█' * (percent // 2) + '-' * (50 - percent // 2)
    print(f"\r进度: |{bar}| {percent}% ({idx}/{total}) {message}", end='', flush=True)
```

### 问题 2：`_is_not_empty` 函数逻辑永远返回 `True`

**文件**：`funcguard/pd_utils/statistics/mask_utils.py`
**行号**：`21-31`
**级别**：🔴 严重bug

**问题描述**：
`_is_not_empty` 函数末尾存在无法到达的 `return True`，且在多个分支中已经分别返回，导致无论传入什么值（包括 `None`、`NaN`、空容器）函数都返回 `True`。这意味着所有使用 `pd_filter(..., ("col", "not empty"))` 的筛选条件都会错误地匹配所有行。

**代码片段**：
```python
def _is_not_empty(x):
    # 处理 numpy/pandas 数组类型
    if isinstance(x, (np.ndarray, pd.Series)):
        return x.size > 0
    # 原生容器优先按长度判断
    if isinstance(x, (str, list, tuple, dict, set)):
        return len(x) > 0
    # 使用 try-except 避免 pd.isna() 在某些类型上的歧义
    try:
        if pd.isna(x):
            return False
    except ValueError:
        return True
    return True  # ← 永远执行不到，且前面已经处理完所有情况后最终也返回 True
```

**修复建议**：
删除末尾多余的 `return True`，改为在 `except` 块中返回 `True`（因为 `pd.isna()` 失败说明值不是 `NaN`，即非空）：
```python
def _is_not_empty(x):
    if isinstance(x, (np.ndarray, pd.Series)):
        return x.size > 0
    if isinstance(x, (str, list, tuple, dict, set)):
        return len(x) > 0
    try:
        if pd.isna(x):
            return False
    except ValueError:
        # pd.isna() 失败说明 x 不是可空的标量值，应视为有值
        pass
    return True
```
