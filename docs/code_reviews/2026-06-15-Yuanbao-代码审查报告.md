### 问题 1：裸 `except:` 捕获所有异常（包括系统异常），难以调试

**文件**：`funcguard/ip_utils.py`
**行号**：`get_public_ip`（约 L25-29）、`is_valid_ip`（约 L46-47）
**级别**：🟠 注意问题

**问题描述**：
`get_public_ip` 中使用了裸 `except:` 来捕获请求异常，内部的 `except:` 同样为裸形式。这会同时捕获 `SystemExit`、`KeyboardInterrupt` 等系统异常，使程序在遇到 `KeyboardInterrupt`（用户按 `Ctrl+C`）时无法正常退出。此外，异常信息仅 `print` 输出，没有堆栈记录，生产环境中错误根因难以追溯。

**代码片段**：
```python
# ip_utils.py
def get_public_ip():
    try:
        # ...
        for service in ip_services:
            try:
                response = requests.get(service, timeout=5)
                # ...
            except:          # <-- 裸 except，隐藏真实错误
                continue
    except Exception as e:
        print(f"获取公网IP地址失败: {e}")  # 只打印，没有堆栈
        return None

def is_valid_ip(ip_string):
    try:
        # ...
    except:    # <-- 裸 except，静默失败
        return False
```

**修复建议**：
1. 将裸 `except:` 改为 `except Exception:`，允许系统异常正常传播：
   ```python
   for service in ip_services:
       try:
           response = requests.get(service, timeout=5)
       except Exception:  # 不捕获 SystemExit/KeyboardInterrupt
           continue
   ```
2. 对于 `is_valid_ip`，裸 `except` 可以改为 `except Exception`，保持当前静默返回 `False` 的行为。

```python
if return_duration == 0:
```

```python
else:  # return_duration == 2
```
